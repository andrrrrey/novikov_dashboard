# Клуб — личный кабинет резидента (backend)

Стек: FastAPI + SQLModel (SQLite на деве). Авторизация email+пароль (JWT).
GetCourse и связанный с ним прогресс в этот скоуп не входят.

## Запуск
```
pip install -r requirements.txt
uvicorn app.main:app --reload
```
При старте создаётся БД, засеваются 9 подсказок, 27 карточек траектории
и администратор (по умолчанию admin@club.ru / admin12345 — сменить через env
ADMIN_EMAIL / ADMIN_PASSWORD). Документация API: http://127.0.0.1:8000/docs

## Тесты
```
pytest -q
```

## Эндпоинты
- POST /auth/login — вход (username=email), возвращает JWT
- GET  /quiz — 9 вопросов (нужна авторизация)
- POST /quiz/submit — ответы -> расчёт уровней и узкого места
- GET  /me/dashboard — узкое место, подсказка, показатели прогресса (Опыт/категории/Знания/Влияние), плашка
- GET/POST/PATCH/DELETE /admin/users — управление резидентами; PATCH принимает `influence` (Влияние)
- GET  /admin/stats — всего резидентов / прошли квиз / не прошли
- GET/PATCH /admin/promo — плашка «Повышайте свой уровень» (заголовок, картинка, ссылка)
- GET/PATCH /admin/getcourse — домен, ключ (маскируется), период опроса, статус, группы
- POST /admin/getcourse/sync — ручной запуск синхронизации в фоне
- GET/PUT /admin/progress-config — какие группы GetCourse входят в уровни категорий («Опыт») и в «Знания»

## GetCourse и показатели прогресса
Сервер сам опрашивает GetCourse по расписанию (`app/getcourse.py`): получает список **всех** групп
(пул для настройки в админке), тянет составы только назначенных групп, сопоставляет резидента по
email и складывает просмотры в `LessonView`. Домен и секретный ключ задаются в админке
(таблица `Setting`), не в env.

Показатели считает `app/progress.py`:
- **Категории** (Менеджмент/Продажи/Маркетинг): стартовый уровень — из теста; прошёл все группы
  уровня (назначенные в админке) → уровень +1, бар обнуляется. `GcAssignment(track="exp", category, level)`.
- **Опыт**: уровень = сумма уровней трёх категорий; бар = общий % пройденных групп опыта;
  «дней на уровне» из `UserStats.exp_level_since` (обновляется при смене суммы).
- **Знания**: плоский бар — доля пройденных групп `GcAssignment(track="know")`.
- **Влияние**: ручные очки `UserStats.influence`, ставит админ.

Новые таблицы (`GcAssignment`, `UserStats`) создаются `create_all` автоматически; колонки к
существующим таблицам не добавляются (не-урочные группы хранятся с `lesson_number=0`).

Планировщик — фоновый asyncio-таск в lifespan (период `gc_poll_hours`, по умолчанию 2 ч). Опрос
укладывается в лимит GetCourse (100 запросов export-API за 2 ч): бюджет запросов + round-robin,
терпеливый опрос экспорта и авто-повтор при «уже запущен один экспорт». В проде нужен **один**
воркер uvicorn (systemd — без `--workers`). В тестах отключается `GETCOURSE_SYNC_ENABLED=0`.
