# Клуб — личный кабинет резидента (backend)

Стек: FastAPI + SQLModel (SQLite на деве). Авторизация email+пароль (JWT).
GetCourse и связанный с ним прогресс в этот скоуп не входят.

## Запуск
```
pip install -r requirements.txt
uvicorn app.main:app --reload
```
При старте создаётся БД, засеваются 9 подсказок, 27 карточек траектории
и администратор (по умолчанию admin@club.ru / admin12345 — сменить через env
ADMIN_EMAIL / ADMIN_PASSWORD). Документация API: http://127.0.0.1:8000/docs

## Тесты
```
pytest -q
```

## Эндпоинты
- POST /auth/login — вход (username=email), возвращает JWT
- GET  /quiz — 9 вопросов (нужна авторизация)
- POST /quiz/submit — ответы -> расчёт уровней и узкого места
- GET  /me/dashboard — уровни, узкое место, подсказка, общий уровень (GetCourse), плашка
- GET/POST/PATCH/DELETE /admin/users — управление резидентами (только админ)
- GET  /admin/stats — всего резидентов / прошли квиз / не прошли
- GET/PATCH /admin/promo — плашка «Повышайте свой уровень» (заголовок, картинка, ссылка)
- GET/PATCH /admin/getcourse — домен, ключ (маскируется), период опроса, статус, группы
- PATCH /admin/getcourse/groups/{id} — включить/выключить учёт группы в уровне
- POST /admin/getcourse/sync — ручной запуск синхронизации в фоне

## GetCourse — прогресс и уровень
Сервер сам опрашивает GetCourse по расписанию (`app/getcourse.py`): читает группы
«Урок NN просмотрен», сопоставляет резидента с аккаунтом по email и складывает просмотры
в таблицу `LessonView`. Уровень 1–10 = `ceil(просмотрено / N_засчитываемых_групп * 10)`
(0 просмотров → 0). Домен и секретный ключ задаются в админке (таблица `Setting`), не в env.

Планировщик — фоновый asyncio-таск в lifespan (период — настройка `gc_poll_hours`, по умолчанию
2 ч). Опрос укладывается в лимит GetCourse (100 запросов export-API за 2 ч): бюджет запросов на
цикл + round-robin по группам. В проде нужен **один** воркер uvicorn (systemd-сервис — без
`--workers`), чтобы планировщик не дублировался. В тестах отключается `GETCOURSE_SYNC_ENABLED=0`.
